# Demo Script

## 2-3 分钟演示路线

1. 打开 Create Case 页面，说明这不是聊天机器人，而是异常运营工作台。
2. 展示默认案例 `PO-2026-004`，解释它同时有金额、数量、状态和关键字段风险。
3. 提交后进入详情页，按顺序讲：
   - 证据快照：采购单、收货单、发票、物流状态
   - 规则命中：系统先用规则定界，不让模型乱猜
   - AI 解释：模型只做摘要、根因整理、SOP 建议
   - 人工确认：涉及高风险动作必须人工确认
   - 审计时间线：每一步都有留痕
4. 提交一条 `approve` 或 `request_more_info` 审核，展示状态变化。

## 一句话总结

这个项目的价值不在于“自动决策”，而在于把企业异常处理中最真实的四件事做对了：证据、规则、边界和留痕。
